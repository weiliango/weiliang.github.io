Frequent Itemsets and Association Rules

Outline:

  • Introduce market-basket model of data
  • Define frequent itemsets
  • Discover association rules
    • Confidence and interest of rules
  • A-Priori Alg. and variations

The Market-Basket Model

  • Really a general many-many mapping(association) between two kinds of things: items and baskets
  • focus on common events, not rare events
    • don't need to focus on identifying all association rules
    • focus on common events, focus pricing strategies or product recommendations

market basket application examples:

  • Plagiarism detection
    • Look for items that appear together in serveral baskets
      • Multiple documents share sentence
  • Identify related concepts in web documents
    • Look for sets of words that appear together in many documents
    • Ignore most common words
  • Drug interactions
    • detect combinations of drugs that result in particular side-effects
    • but requires extension: absence of an item needs to be observed as well as presence

Define Frequent Itemsets

Support for itemset I = the number of baskets containing all items in I

Given a support threshold s, sets of items that appear in at least s baskets are called frequent item sets

Association Rules

Rule ${i_{1}, i_{2}, …, i_{k}} \to j$ means: if a basket contains all of $i_{1}, i_{2}, …, i_{k}$, then it is likely to contain j

Confidence

Ratio of support for $I \cup {j}$ with support for I

$\to$ want to identify association rules with high confidence

but, not all high-confidence rules are interesting

$\to$ interest of an association rule $I \to j$: difference between its confidence and the fraction of baskets that contain j

  • High positive/negative interest means presence of $I$ encourages or discourages presence of $j$
  • interesting rules are those with high positive or negative interest values(usually above 0.5)

Finding Useful Association Rules

target: find all association rules with $support \ge s$ and $confidence \ge c$

if ${i_{1}, i_{2}, …, i_{k}} \to j$ has high support and confidence, then both ${i_{1}, i_{2}, …, i_{k}}$ and ${i_{1}, i_{2}, …, i_{k}, j}$ will be frequent

Find Frequent Itemsets

computation model

  • typically, market basket data are kept in flat files rather than in a database
  • use k nested loopt to generate all sets of size k
  • $generate \to count \to frequent Itemsets$
  • boundaries between baskets are -1
  • measure the cost by the number of passes an algorithm takes

Bottleneck

  • memory $\to$ hardest problem is to find the frequent pairs
  • naive alg. $\binom{n}{2}$, fails if $items^2$ exceeds main memory
    • two approaches
      • count all pairs, using a triangular matrix
      • keep a table of

Algorithm

  • PCY
    • we cannot renumber the items in PCY, it's random, so we cannot use Triangular
    • before first Pass, use the memory to hash the pairs and use a bit map for optimize
  • Multistage
    • after Pass 1 of PCY, rehash only those pairs that qualify for Pass 2 of PCY
    • need check every Bitmap
    • two hashfunction has to be independent
  • Son
    • 就是利用单调性，如果一个item freqent，那它至少在一个子集里面frequent
    • 用mapreduce，把至少在一个子集里frequent的作为candidate
    • 第二个phase，再跑一遍测试
  • Toivonen
    • 没有frequent，会死循环
    • 构建negative border，如果里面有frequent，重新sampling
