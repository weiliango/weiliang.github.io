Mining Social-Network Graph

Goal: find densely linked clusters

What can we gain:

  • topic analysis
  • photo clustering
  • improved recommendation methods

some concepts:

  • node degree

$$ deg(V_{i}) = k_{i} = number \ of \ neighbors $$

  • adjacency matrix $$ A_{ij} = link \ between\ nodes i and j $$

    $$ 0\to no\ link\ 1 \to link \ \alpha \to link \ with \ weight \ equal \ to \ \alpha $$

  • modularity: quantifies the extent to which a given graph partition into communities presents a systematic tendency to have more intra-community links than the same community structure would present if the links would be rewired under ER(ERDOS-RENYI) graph model. $$ Q=\frac{1}{2m}\sum_{i,j}(A_{ij}-\frac{k_{i}k_{j}}{2m})\delta(c_{i}, c_{j}) $$ tip:

    • random graph(ER MODEL), expect Q=0
    • typically $0.3\to0.7$

  • betweenness: number of shortest paths passing over the edge

Girvan-Newman Algorithm

  1. start with one cluster(the social network) and recursively split it
    1. based on the notion of edge betweenness
  2. Girvan-Newman Algorithm(undirected unweighted networks):
    1. visits each node X once
    2. computes the number of shortest paths from X to each of the other nodes that go through each of the edges
      1. perform a bfs starting at node X
        1. level of each node is the shortest path from X to that node
        2. so edges on the same level can never be part of a shortest path from X
        3. edges between levels are called DAG edges(DAG = Directed Acyclic Graph)
          1. each DAG edge is part of at least one shortest path from root X
      2. label each node by the number of shortest paths that reach it from the root node
      3. calculate for each edge e the sum over all nodes Y of the fraction of the shortest paths from the root X to Y that go through edge e
        1. assign node and edge values starting from bottom
  3. Repeat
    1. Calculate betweenness of edges
      1. thresholding to remove high between edges or
      2. remove edges with highest betweenness
  4. complete
    1. repeat for every node as root
    2. sum the contributions on each edge
    3. divide by 2 to get true betweenness
      1. every shortest path will be counted twice, once for each of its endpoints

$\to$ give a hierarchical decomposition of the network

How to select the number of clusters?

Modularity Q: a measure of how well a network is partitioned into communities

Null Model: Configuration Model（指的就是degree相同，但是没有边，所有边都是随机排列，概率意义上的model——大概，不准确）

The expected number of edges in multigraph G`: $$ \frac{1}{2}\sum_{i,j \in N}\frac{k_{i}k_{j}}{2m} = \frac{1}{2}\times \frac{1}{2m} \times \sum_{i \in N}k_{i} \times \sum_{j \in N} k_{j} = \frac{1}{4m} \times 2m \times 2m = m $$ Modularity of partitioning S of graph G: $$ Q = \sum_{s \in S} [edges \ within \ s \ - \ expected \ edges \ within \ group \ s] $$ hint: s is a cluster, S is clusters $$ Q(G, S) = \frac{1}{2m}\sum_{s \in S}\sum_{i \in s}\sum_{j \in s}(A_{ij}-k_i\times \frac{k_{j}}{2m}) $$ $\frac{1}{2m}$ normalizing to : -1 < Q < 1

0.3-0.7 < Q means significant community structure

Graph Partitioning

good partition:

  • cut is minimized(edges used in this cut)
  • two sets approximately equal in size
  • maximum the number of within-group connections
  • minimize the number of between-group connections

Using normalized-cut to find Good Graph Partitions

$$ cut(A, B) = \sum_{i \in A, j \in B} W_{ij} $$

Normalized-cut: $$ ncut(A,B) = \frac{cut(A,B)}{vol(A)} + \frac{cut(A,B)}{vol(B)} $$ vol(A) - total weight of the edges with at least one endpoint (avoid the minimum cut just has one node, so that vol(A) is 0) in A

Using Matrix Algebra to find Good Graph Partitions

Laplacian matrix(L):

  • x = (1, … , 1) then $L\times x = 0$, so $\lambda=\lambda_{1} =0$
    • Eigenvalues are non-negative real numbers
    • Eigenvalues are real and orthogonal
  • second-smallest eigenvalue it the minimum of $x^TLx$ where x = [x1,x2,….,xn]
  • sum of $x_{i}^2 = 1$
  • this second smallest eigenvector x will have some positive and negative components -> make the result is 0
  • Partition the graph by taking one set to be the nodes I whose corresponding vector component xi is positive

What is the meaning of min $x^TLx$ on G? $$ \sum_{i,j \in E}(x_{i}-x_{j})^2 $$ what else do we know about x?

  • x is unit vector: $\sum_{i}x_{i}^2 = 1$
  • x is orthogonal to [1,…, 1] thus:
    • $\sum_{i}x_{i} \times 1 = \sum_{i}x_{i} = 0$

Remember

​ we want to assign values x_{i} to nodes i such that few edges cross 0 $$ \lambda_{2} = min \frac{\sum_{(i,j)\in E}(x_i - x_j)^2}{\sum_{i}x_{i}^2} $$

Spectral Clustering Algorithms

  1. Pre-processing:
    1. Build Laplacian matrix L of the graph
  2. Decomposition
    1. Find eigenvalues and eigenvectors
    2. Map vertices to corresponding components of $\lambda_{2}$
  3. Grouping
    1. Sort components of reduced 1-dimensional vector
    2. identity clusters by splitting the sorted vector in two

How to choose a splitting point?

  1. split at 0 or a median value or
  2. use normalized cut

k-Way Spectral Clustering

two approaches:

  • Recursive bi-partitioning
    • inefficient, unstalbe
  • Cluster multiple eigenvectors
    • prevent instability due to information loss
